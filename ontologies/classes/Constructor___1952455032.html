Annotations (1)

  • comment "Constructors are used to create an instance of a class in object-orientation. In some languages, a constructor is a special method. However, e.g., Java Interfaces and Annotations cannot have constructors. Further a constructor's signature often differs from a method's one. Constructor signatures can only have access modifiers, i.e., they cannot be 'abstract', 'final', 'native', 'static', or synchronized. They cannot return anything and they need to have the same identifier as the class they're defined in. Because of this differences, we did not model Constructor as a subclass of Method." ()
